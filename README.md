# Лабораторная работа: Моделирование информационной системы университета

## 1. Диаграмма вариантов использования (Use Case)
Описывает функциональные возможности системы для различных ролей: **Декан**, **Сотрудник администрации**, **Научный сотрудник** и **Преподаватель**.

![Use Case Diagram](diagr%20png/use%20caase.PNG)

---

## 2. Диаграмма классов (Class Diagram)
Спроектирована архитектура системы:
- Использовано наследование (`Employee` -> `Dean`, `Lecturer`, `ResearchAssociate`).
- Реализована композиция (Университет -> Факультеты -> Институты).
- Применен ассоциативный класс `Participation` для связи сотрудников с проектами.

![Class Diagram](diagr%20png/диагр%20классов.PNG)

---

## 3. Диаграмма последовательности (Sequence Diagram)
Демонстрирует сценарий **добавления нового научного сотрудника**:
1. Декан отправляет запрос университету.
2. Университет находит факультет.
3. Факультет поручает кафедре создать сотрудника.
4. Кафедра создает объект сотрудника и подтверждает успех.

![Sequence Diagram](diagr%20png/Sequence%20.PNG)

---

## 4. Диаграмма состояний (State Machine Diagram)
Показывает жизненный цикл научного сотрудника (`ResearchStaff`):
- **Создан** → **Назначен на кафедру** → **Работает над проектами** → **Уволен (Неактивен)**.

![State Machine Diagram](diagr%20png/диагр%20сосст.PNG)
